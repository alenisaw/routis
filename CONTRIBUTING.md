# Contributing

Routis is a small early-stage project. The repo is still settling on its first implementation steps, so changes should stay narrow and practical.

## Before You Start

- read the existing docs first
- keep scope small
- avoid adding dependencies unless they are clearly needed

## Branch Names

Examples:

- `feat/cli-router`
- `feat/policy-loader`
- `fix/policy-defaults`
- `docs/readme-cleanup`

## Commits

Keep commit messages direct. Examples:

- `docs: clarify project scope`
- `feat(cli): add route selection command`
- `feat(policy): load default preset`

Single-purpose commits are preferred.

## Pull Requests

A good pull request here usually:

- explains what changed
- stays small enough to review quickly
- avoids unrelated cleanup
- updates docs when needed

If a change does not help routing, token economy, explainability, or the Codex workflow, it is probably outside the current scope.
